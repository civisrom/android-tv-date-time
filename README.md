# Android TV Time Fixer

**Исправление проблем с синхронизацией времени на Android TV**

## Описание проблемы

Многие телевизоры и Android TV-приставки, особенно в регионах с сетевыми ограничениями, сталкиваются с проблемой сброса системных часов после отключения от электросети. Несмотря на включенную функцию автоматической синхронизации времени, устройство не может подключиться к серверу времени, что приводит к следующим последствиям:

*   **Потеря доступа к интернет-приложениям:** Многие приложения требуют точного времени для корректной работы.
*   **Необходимость ручной установки времени:** Пользователю приходится вручную выставлять время каждый раз после отключения устройства от питания.
*   **Сообщение "Подключено, без доступа к интернету" в настройках Wi-Fi:** Это индикатор того, что устройство не может синхронизировать время с сервером.

**Причина:** Основной причиной является невозможность подключения устройства к стандартному NTP-серверу Google (`time.android.com`) из-за сетевых ограничений в указанных регионах.

**Решение:** Android TV Time Fixer позволяет решить эту проблему, заменяя стандартный NTP-сервер Google на альтернативный, доступный в вашем регионе.

## О программе

**Android TV Time Fixer** — это кроссплатформенная утилита для Windows, Linux и macOS, предназначенная для управления настройками NTP-сервера на устройствах Android TV через ADB (Android Debug Bridge).

## Установка

### Windows

1.  Скачайте архив `AndroidTVTimeFixer-windows.zip` из раздела [Releases](https://github.com/civisrom/android-tv-date-time/releases).
2.  Распакуйте архив в удобное место на вашем компьютере, например, `D:\AndroidTVTimeFixer`.
3.  Запустите `AndroidTVTimeFixer.exe` или используйте `start.bat` / `start.ps1`.

Запуск через **PowerShell**

1.  Откройте **PowerShell** от имени администратора.
2.  Перейдите в папку с программой:
    ```powershell
    cd "D:\AndroidTVTimeFixer"
    ```
3.  Запустите программу:
    ```powershell
    .\AndroidTVTimeFixer.exe
    ```

### Linux

1.  Скачайте архив `AndroidTVTimeFixer-linux.zip` из раздела [Releases](https://github.com/civisrom/android-tv-date-time/releases).
2.  Распакуйте архив:
    ```bash
    unzip AndroidTVTimeFixer-linux.zip
    ```
3.  Сделайте файл исполняемым и запустите:
    ```bash
    chmod +x AndroidTVTimeFixer
    ./AndroidTVTimeFixer
    ```

### macOS

1.  Скачайте архив `AndroidTVTimeFixer-macos.zip` из раздела [Releases](https://github.com/civisrom/android-tv-date-time/releases).
2.  Распакуйте архив и запустите приложение.

## Подготовка Android TV

### Включение отладки ADB (режим разработчика)

1.  Откройте на вашем Android TV: **Настройки** > **Настройки устройства** > **Об устройстве**.
2.  Нажмите 7 раз на пункт **"Сборка"**, чтобы разблокировать режим разработчика.
3.  Перейдите: **Настройки устройства** > **Для разработчиков**.
4.  Включите **"Отладка по сети"**.
5.  Откройте: **Настройки** > **Дата и время**.
6.  Включите: **Автонастройка даты и времени** > **Использовать время сети**.
7.  Для повышения безопасности рекомендуется отключить режим разработчика после завершения настройки NTP-сервера.

## Как пользоваться программой

### Пункт 1 — Изменить сервер времени NTP по коду страны

Программа запрашивает IP-адрес вашего устройства (телевизора или приставки), подключается к нему по ADB, показывает текущие настройки и предлагает ввести двухбуквенный код страны (например, `ru`, `ua`, `de`). Можно ввести `?название` для поиска страны по названию. После выбора кода программа автоматически установит NTP-сервер соответствующего региона.

> **Подсказка:** Если вы не знаете код своей страны, сначала откройте **пункт 3** — там отображается полный список кодов стран с названиями и NTP-серверами. Скопируйте нужный код в буфер обмена и вставьте его при запросе в пункте 1.

### Пункт 2 — Изменить сервер времени NTP на пользовательский

Аналогично пункту 1, но вместо кода страны вы вводите адрес NTP-сервера вручную (доменное имя или IP-адрес). Программа проверит корректность формата и установит указанный сервер на устройство.

> **Подсказка:** Адрес NTP-сервера можно взять из **пункта 3** (серверы по странам) или **пункта 4** (альтернативные серверы — Cloudflare, Google и др.). Откройте нужный пункт, скопируйте адрес сервера в буфер обмена и вставьте его при запросе в пункте 2.

### Пункт 3 — Показать коды стран с названиями и NTP-серверами

Выводит полный список поддерживаемых кодов стран (65+), их названия и соответствующие NTP-серверы. Можно использовать интерактивный поиск. Результаты можно скопировать в буфер обмена.

> **Использование результатов:** Скопированный код страны можно вставить в **пункте 1**, а скопированный адрес NTP-сервера — в **пункте 2** при ручной настройке.

### Пункт 4 — Показать доступные альтернативные NTP-серверы

Показывает список альтернативных NTP-серверов: региональные пулы, Cloudflare, Google и другие. Результаты можно скопировать в буфер обмена.

> **Использование результатов:** Скопированный адрес сервера можно вставить в **пункте 2** для ручной установки на устройство.

### Пункт 5 — Показать текущую информацию об устройстве

Подключается к устройству и выводит подробную информацию: модель, производитель, версия Android, серийный номер, CPU, RAM, разрешение экрана, сетевые параметры (IP, MAC), текущий NTP-сервер, часовой пояс, аптайм, а также сравнение времени устройства и ПК.

### Пункт 6 — Пинговать NTP-серверы

Проверяет доступность всех известных NTP-серверов (110+). Показывает время отклика (RTT), процент успешных подключений, сортирует результаты по доступности и скорости.

### Пункт 7 — Управление серверами

Открывает подменю для работы с избранными серверами:
- **Показать избранные** — список сохранённых серверов
- **Добавить текущий сервер** — сохраняет текущий NTP-сервер устройства в избранное
- **Копировать сервер** — копирует текущий сервер устройства в буфер обмена
- **Вставить сервер** — устанавливает на устройство сервер из буфера обмена
- **Удалить сервер** — удаляет выбранный сервер из списка избранного
- **Пинговать NTP-серверы** — проверка доступности серверов
- **Экспорт / Импорт настроек** — сохранение и восстановление всех настроек (язык, IP, избранные серверы) в JSON-файл

### Пункт 8 — Сканирование сети и групповое обновление NTP

Открывает подменю для работы с несколькими устройствами:
- **Сканировать сеть** — автоматически находит устройства Android TV в локальной сети по открытому ADB-порту 5555
- **Подключиться к найденному устройству** — выбрать и подключиться к одному из обнаруженных устройств
- **Групповое обновление NTP** — установить NTP-сервер сразу на все найденные или вручную введённые устройства
- **Статус синхронизации времени** — сравнить время устройства с временем ПК

### Пункт 9 — Автоматическая установка NTP-сервера (экспериментальный режим)

Полностью автоматический режим:
1. Сканирует локальную сеть и находит устройства Android TV
2. Подключается к выбранному устройству
3. Определяет ваш регион по часовому поясу системы
4. Проверяет все доступные NTP-серверы (110+) и измеряет скорость отклика
5. Показывает Топ-5 самых быстрых серверов с RTT, процентом успешности и смещением
6. Автоматически рекомендует лучший сервер
7. Устанавливает выбранный сервер на устройство

### Пункт 10 — Режим терминала

Интерактивный режим для выполнения любых ADB и системных команд. Полезен для продвинутых пользователей:
- Управление приложениями (установка, удаление)
- Передача файлов (push/pull)
- Снимки экрана и запись видео
- Диагностика системы
- Перезагрузка устройства

Команды: `help` — справка, `clear` — очистка экрана, `exit` — выход из терминала.

### Пункт 0 — Выход

Завершение работы программы.
